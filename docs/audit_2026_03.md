## SECURITY

### SEC-001 — SSRF via unconstrained URL scraping [CRITICAL / FIXED]
**File:** `backend/app/pipeline/research/scraper.py`
The scraper fetches any URL returned by search providers without filtering private/internal IP ranges.
**Fix applied:** `_is_safe_url()` added — resolves hostname and rejects loopback, RFC-1918, link-local, reserved IPs, and non-http/https schemes before any request is issued. 7 tests added.

### SEC-002 — Rate limit bypass via X-Forwarded-For spoofing [HIGH / TODO]
**File:** `backend/app/api/deps.py:62-66`
Rate limit key derived from `x-forwarded-for` with no trusted-proxy validation — any client can rotate the header.
**Fix:** Add `TRUSTED_PROXIES` setting; only trust `x-forwarded-for` if `request.client.host in settings.trusted_proxies`.

### SEC-003 — LLM prompt injection via user-controlled input [CRITICAL / FIXED]
**File:** `backend/app/pipeline/writer/section_writer.py`
`topic`, `discipline`, `additional_instructions` interpolated directly into LLM system prompts via `.format()`, causing `KeyError` on `{var}` patterns and potential structural injection.
**Fix applied:** `_safe(text)` helper escapes `{` → `{{` and `}` → `}}` on all user-controlled fields before `.format()` interpolation. 3 tests added.

### SEC-004 — Robokassa webhook crashes on non-integer InvId [HIGH / FIXED]
**File:** `backend/app/api/routes/payments.py:126`
`int(inv_id)` raises `ValueError` on malformed input, returning HTTP 500.
**Fix:** Wrap in `try/except ValueError`, return `"bad inv_id"` string.

### SEC-005 — File upload validates only extension, not content [HIGH / TODO]
**File:** `backend/app/api/routes/jobs.py:190`
`file.filename.endswith(".docx")` is client-controlled. No MIME/magic-byte check.
**Fix:** After reading bytes, verify `content[:2] == b'PK'` (ZIP magic bytes for .docx).

### SEC-006 — Hardcoded default credentials [HIGH / FIXED]
**Files:** `backend/app/config.py:50-51`, `infra/docker-compose.yml:9-10`
`s3_access_key = "minioadmin"`, `POSTGRES_PASSWORD: courseforge` are hardcoded.
**Fix:** Remove Python defaults for S3 creds; use `${VAR}` env substitution in compose.

### SEC-007 — Tavily API key sent in request body [LOW / TODO]
**File:** `backend/app/pipeline/research/searcher.py:36-40`
API key in body appears in HTTP logs and APM traces.
**Fix:** Move to `Authorization: Bearer` header.

### SEC-008 — Timing-attack-vulnerable signature comparison [LOW / FIXED]
**File:** `backend/app/services/robokassa.py:65`
`==` is not constant-time.
**Fix:** Use `hmac.compare_digest()`.

### SEC-009 — CORS wildcard + credentials in debug mode [LOW / TODO]
**File:** `backend/app/main.py:82-88`
`allow_origins=["*"]` + `allow_credentials=True` — contradicted by CORS spec but passes server-side.

## BUGS

### BUG-001 — Race condition: credits can go negative [CRITICAL / TODO]
**File:** `backend/app/api/routes/jobs.py:87-95`
Two concurrent requests both read `credits_remaining = 1`, both pass the check, both deduct.
**Fix:** Replace read-check-update with atomic `UPDATE users SET credits_remaining = credits_remaining - 1 WHERE id = ? AND credits_remaining > 0`; check `rowcount == 0` for 402.

### BUG-002 — Cancelled jobs overwritten by worker as "completed" [HIGH / TODO]
**File:** `backend/app/workers/tasks.py:226`
Worker continues after job is cancelled via API; overwrites status to COMPLETED.
**Fix:** In `run_pipeline`, after each DB block, check `if job.status == JobStatus.CANCELLED: return`.

### BUG-003 — `result.document_bytes` None raises TypeError [MEDIUM / TODO]
**File:** `backend/app/pipeline/orchestrator.py:334`
`len(result.document_bytes)` raises `TypeError` if formatter returns None.
**Fix:** Add `if not result.document_bytes: raise PipelineError(...)`.

### BUG-004 — Magic number sentinel `99` for conclusion chapter [MEDIUM / FIXED]
**Files:** `section_writer.py:219`, `orchestrator.py:515`
Sentinel `99` and `0` for chapter numbers are hidden contracts with no named constants.
**Fix:** Add `CHAPTER_INTRO = 0`, `CHAPTER_CONCLUSION = 99` to `shared/schemas/pipeline.py`.

### BUG-005 — Default user created with race condition under concurrent load [MEDIUM / TODO]
**File:** `backend/app/api/routes/jobs.py:225-236`
Two concurrent requests can both find no "default" user and both insert.
**Fix:** (Superseded by ARCH-001/004 fix — real user lookup by telegram_id eliminates "default" user entirely.)

### BUG-007 — `datetime.utcnow()` deprecated in Python 3.12 [MEDIUM / FIXED]
**Files:** `tasks.py:131,152,229`, `routes/jobs.py:164`, `routes/payments.py:139`
`datetime.utcnow()` returns naive datetimes; deprecated since Python 3.12.
**Fix:** `from datetime import UTC; datetime.now(UTC)`.

### BUG-008 — Case-sensitive `.docx` extension check [LOW / FIXED]
**File:** `backend/app/api/routes/jobs.py:190`
Rejects `Template.DOCX`.
**Fix:** `file.filename.lower().endswith(".docx")`.

### BUG-009 — arq task name as string literal [LOW / TODO]
**File:** `routes/jobs.py:114`
`await arq_pool.enqueue_job("run_pipeline", ...)` — renaming the function causes silent failures.
**Fix:** `run_pipeline.__name__` or a shared constant.

## ARCHITECTURE

### ARCH-001 / ARCH-004 — Credit system non-functional: single "default" user [CRITICAL / TODO]
**Files:** `routes/jobs.py:83-84`, `routes/payments.py:189-196`
- Jobs route always uses `User(username="default")`, ignoring telegram_id
- Payments route creates/uses `User(telegram_id=...)` — different DB rows
- Credits paid by a real user → their telegram_id user row
- Credits deducted when creating a job → the "default" user row
- Result: paying users can never use their credits for jobs

**Fix:**
1. Add `telegram_id: int` to `JobCreate` schema
2. In `create_job`, call `_get_or_create_user_by_telegram(db, job_in.telegram_id)` instead of `_get_or_create_default_user`
3. In `bot/app/handlers/generate.py`, pass `telegram_id=callback.from_user.id` when building `JobCreate`

### ARCH-002 — API route imports private worker function [HIGH / FIXED]
**File:** `routes/jobs.py:198-199`
`from backend.app.workers.tasks import _upload_document_to_s3` — layering violation.
**Fix:** Created `backend/app/services/storage.py` with public `upload_to_s3` / `download_from_s3`; both routes and workers import from there.

### ARCH-003 — vision_llm typed as concrete class, not interface [HIGH / TODO]
**Files:** `orchestrator.py:88-92`, `visual_matcher.py:108-113`
`vision_llm: OpenRouterProvider` breaks the factory abstraction.
**Fix:** Add `generate_with_vision` abstract method to `LLMProvider`; update concrete classes; change type hints to `LLMProvider`.

### ARCH-005 — `get_settings()` called directly, bypassing FastAPI DI [MEDIUM / TODO]
**Files:** `deps.py:56`, `routes/payments.py:110`, `tasks.py:140`
`lru_cache` makes settings globally cached at module load time, complicating tests.
**Fix:** Use `settings: Settings = Depends(get_settings)` throughout dependency chain.

### ARCH-006 — No API versioning [LOW / TODO]
All routes at `/api/...` with no version prefix. Bot and backend can't evolve independently.
**Fix:** Add `/api/v1/` prefix; add deprecation shim at `/api/`.

### ARCH-007 — Pipeline config in shared schemas (backend-internal concern) [LOW / TODO]
**File:** `shared/schemas/pipeline.py`
`PipelineConfig` with LLM model names etc. should be internal to backend.
**Fix:** Move to `backend/app/pipeline/config.py`.

## PERFORMANCE

### PERF-001 — New httpx.AsyncClient per LLM/search call [HIGH / TODO]
**Files:** `openrouter.py:51`, `anthropic.py`, `searcher.py:31`
New TCP + TLS handshake on every LLM call. With 20-50 LLM calls per pipeline, adds seconds.
**Fix:** Class-level `AsyncClient` with connection pooling; manage lifecycle via FastAPI lifespan.

### PERF-002 — `list_jobs` has no user filter — full table scan [HIGH / TODO]
**File:** `routes/jobs.py:131-146`
No `WHERE user_id = ?`. Degrades for all users as DB grows.
**Fix:** After ARCH-001/004 fix, add user filter + index on `(user_id, created_at DESC)` via new Alembic migration.

### PERF-003/004 — Visual matcher generates doc and re-renders reference redundantly [MEDIUM / TODO]
**File:** `visual_matcher.py:131,181,247-251`
Reference doc rendered twice; output doc generated `max_iterations + 1` times instead of `max_iterations`.
**Fix:** Cache `ref_images`; return `doc_bytes` from last iteration; remove final unconditional generation.

### PERF-005 — Full scraped text stored in JSONB (huge rows) [MEDIUM / TODO]
**File:** `models/job.py:45-47`
`research_data` stores full source text — potentially 75 MB per job row.
**Fix:** Move `full_text` to S3; store only metadata + S3 key in JSONB.

### PERF-006 — All 8 sources sent to every section regardless of relevance [MEDIUM / TODO]
**File:** `writer/section_writer.py:248-252`
12,000 chars of repeated context per section × 12 sections = 144,000 chars wasted.
**Fix:** Filter sources by chapter relevance before formatting section prompts.

### PERF-007 — boto3 S3 client created on every upload [MEDIUM / FIXED]
**File:** `workers/tasks.py:35-45`
New connection per upload.
**Fix:** Module-level cached client via `functools.lru_cache`.

### PERF-008 — User model loads all jobs eagerly [LOW / TODO]
**File:** `models/user.py:47`
`lazy="selectin"` runs a second SELECT for all jobs on every User load.
**Fix:** Change to `lazy="noload"`; add explicit `selectin` only where needed.

## TESTING

### TEST-001 — No tests for bot FSM handlers [MEDIUM / TODO]
`bot/tests/` has no handler tests. The `generate.py` FSM flow (topic → submit) has zero coverage.

### TEST-002 — No integration test for API → worker → DB flow [MEDIUM / TODO]
No test covers `POST /jobs` → arq enqueue → `run_pipeline` → DB updated.

### TEST-003 — Payment webhook idempotency test [MEDIUM / FIXED]
Added: call webhook twice with same inv_id; assert credits incremented exactly once.

### TEST-004 — MockLLMProvider silently returns `""` when responses exhausted [MEDIUM / FIXED]
**File:** `backend/app/testing.py:37-40`
**Fix:** Raises `AssertionError` with descriptive message when call index exceeds configured responses.

### TEST-005 — No retry on transient LLM/search API failures [HIGH / TODO]
If OpenRouter returns 503, pipeline fails immediately. Credit consumed, user has no recourse.
**Fix:** `tenacity`-based retry with exponential backoff on 429/502/503 in `openrouter.py:generate` and `searcher.py:search`.

### TEST-006 — No circuit breaker for LLM failures [MEDIUM / TODO]
All 3 workers could block for 900 seconds on downed LLM provider.
**Fix:** Circuit breaker (e.g., `pybreaker`) that opens on consecutive failures.

### TEST-007 — No dependency vulnerability scanning in CI [LOW / TODO]
No `pip-audit`, `safety`, or `trivy` in `.github/workflows/ci.yml`.

### TEST-008 — Health check doesn't verify actual service connectivity [HIGH / FIXED]
**Fix:** Added async DB ping (`SELECT 1`) and Redis `PING` to `/api/health/ready`; returns 503 if either fails.

## INFRASTRUCTURE

### INFRA-001 — All containers run as root [MEDIUM / FIXED]
**Fix:** Added `RUN adduser` + `USER app` to all three Dockerfiles.

### INFRA-002 — No `.dockerignore` [MEDIUM / FIXED]
**Fix:** Created `.dockerignore` excluding `.git`, `__pycache__`, `.env`, test data, etc.

### INFRA-003 — No pip lockfile — non-reproducible builds [MEDIUM / TODO]
Builds a month apart may install different patch versions.
**Fix:** `uv lock` or `pip-compile`; use lockfile in Dockerfiles.

### INFRA-004 — Multi-stage build label declared but unused [LOW / TODO]
`FROM python:3.12-slim AS base` — the `AS base` alias is never referenced by a second stage.

### INFRA-005 — Hardcoded credentials in docker-compose.yml [MEDIUM / FIXED]
**Fix:** Replaced inline values with `${POSTGRES_PASSWORD}`, `${MINIO_ROOT_USER}`, `${MINIO_ROOT_PASSWORD}` env substitution.

### INFRA-006 — MinIO has no healthcheck [LOW / TODO]
Worker may start processing jobs before MinIO is ready.
**Fix:** Add `healthcheck` to minio service; add `depends_on: minio: condition: service_healthy` to backend/worker.

### INFRA-007 — No pip caching in CI [LOW / TODO]
Every run downloads all deps from scratch.
**Fix:** Add `actions/cache@v4` for pip in CI workflow.

### INFRA-009 — LibreOffice in backend image unnecessarily [LOW / TODO]
LibreOffice (~800 MB) needed only by worker for rendering; backend image shouldn't carry it.

### INFRA-010 — No resource limits in docker-compose [LOW / TODO]
A runaway pipeline can consume all host RAM/CPU.
**Fix:** Add `mem_limit: 2g`, `cpus: "1.0"` to worker service.
