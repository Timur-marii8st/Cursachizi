# Pipeline Quality Fixes — План исправления качества генерации

**Дата:** 2026-03-09
**Контекст:** Анализ реальной сгенерированной курсовой выявил системные проблемы с источниками, библиографией и форматированием.

---

## Приоритет 1: Система источников и библиографии (КРИТИЧНО)

### Проблема
1. LLM галлюцинирует источники — генерирует правдоподобные, но несуществующие книги
2. Реальные источники из research-стадии (scraped URLs/titles) не маппятся в финальную библиографию
3. Ссылки [1][2][3] в тексте нумеруются локально для каждой секции, не привязаны к единому списку
4. В конце каждой секции пустой блок "Библиографический список:" без содержания
5. Итоговый список литературы содержит всего 3 фейковых источника вместо 15-20 реальных

### Решение
- **Шаг 1**: После research+ranking — сформировать единый `BibliographyRegistry` (пронумерованный список всех источников)
- **Шаг 2**: Передавать в каждую секцию полный registry + указание какие источники релевантны для данной секции
- **Шаг 3**: В промпте section_writer — требовать ссылки ТОЛЬКО по номерам из registry, НЕ выдумывать свои
- **Шаг 4**: Убрать из промпта требование генерировать "Библиографический список" в конце секции
- **Шаг 5**: В `docx_generator` — генерировать единый "СПИСОК ИСПОЛЬЗОВАННЫХ ИСТОЧНИКОВ" из registry
- **Шаг 6**: Post-processing — валидировать что все [N] в тексте ссылаются на существующие записи в registry

### Файлы для изменения
- `shared/schemas/pipeline.py` — добавить `BibliographyEntry`, `BibliographyRegistry`
- `backend/app/pipeline/orchestrator.py` — создать registry после research, передать в writer
- `backend/app/pipeline/writer/section_writer.py` — переработать промпты, убрать блок библиографии
- `backend/app/pipeline/formatter/docx_generator.py` — генерировать список литературы из registry

## Приоритет 2: Дубли и форматирование

### 2a. Дубль заголовка "ВВЕДЕНИЕ"
- `docx_generator` добавляет заголовок "ВВЕДЕНИЕ"
- `section_writer` в промпте пишет "Напиши ТОЛЬКО текст введения" но LLM иногда начинает с "ВВЕДЕНИЕ"
- **Решение**: strip заголовок из начала текста введения перед вставкой в документ

### 2b. Двойные кавычки «« »»
- Пользователь вводит `«Менеджмент»`, шаблон добавляет свои `«»`
- **Решение**: strip `«»` из пользовательского ввода при подстановке в шаблонные поля

### 2c. Оглавление
- В документе текст `[Обновите оглавление: ПКМ → Обновить поле]`
- **Решение**: генерировать оглавление программно из заголовков, а не через Word-поля

## Приоритет 3: Качество контента

### 3a. Отсутствие конкретики
- Нет кейсов реальных компаний, продуктов, законов
- **Решение**: в промпт добавить: "Приведи конкретные примеры российских компаний и продуктов"

### 3b. Повторы между секциями
- Одни и те же мысли перефразируются
- **Решение**: coherence_checker (уже запланирован в audit plan) + передавать summary предыдущих секций

### 3c. Нет визуализации
- Ни одной таблицы или рисунка в 25-страничной работе
- **Решение**: в промпт для ключевых секций добавить требование таблицы сравнения

## Порядок реализации

1. **BibliographyRegistry** — схемы + маппинг источников (shared/schemas/pipeline.py)
2. **Промпты section_writer** — убрать блок библиографии, передать registry
3. **docx_generator** — единый список литературы, fix дублей заголовков
4. **Post-processing** — валидация ссылок [N], strip лишних кавычек
5. **Тесты** — для каждого изменения
